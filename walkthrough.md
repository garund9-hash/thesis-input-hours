# 논문 작성 시간 트래킹 대시보드 (Thesis Time Tracker) 완성 및 워크스루

사용자님의 피드백(로컬 저장 방식, 다크/라이트 모드, 메모 기능 추가)을 완벽하게 반영하여 논문 투입 시간 관리 대시보드 애플리케이션 개발을 완료했습니다.

## 🛠️ 주요 변경 및 구현 사항

1. **프로젝트 환경 구성**
   - React + Vite 환경을 구축하고, 파이 차트 표시를 위해 `recharts` 패키지를 설치했습니다.
   - 사용자 경험 강화를 위한 아이콘 라이브러리(`lucide-react`)를 적용했습니다.

2. **UI/UX 디자인 (고도화된 Vanilla CSS)**
   - 시인성과 가독성이 뛰어난 프리미엄 모던 웹 디자인 트렌드(Glassmorphism)를 적용했습니다.
   - 우측 상단 해/달 아이콘 버튼을 통해 **다크 모드와 라이트 모드를 자유롭게 전환**할 수 있습니다. 

3. **로컬 스토리지 기반 데이터 관리 (`useThesisData` Hook)**
   - **카테고리 관리**: '연구배경 작성', '선행연구 분석' 등 본인만의 커스텀 논문 작업 카테고리를 색상과 함께 직접 추가/삭제할 수 있습니다.
   - **시간 기록**: 작업 날짜, 카테고리, 투입 시간(시간 단위), 그리고 **작업 내용을 적을 '메모'** 필드를 통합했습니다.
   - 이 모든 데이터는 브라우저의 로컬 스토리지(Local Storage)에 안전하게 영구 저장되어 서버가 필요 없습니다.

4. **대시보드 뷰 구성**
   - **핵심 지표 요약 카드**: 총 투입 시간, 가장 많은 시간을 투자한 카테고리, 전체 기록 횟수가 상단에 시원하게 표시됩니다.
   - **데이터 시각화 차트**: Recharts 라이브러리를 활용해 카테고리별 누적 시간 배분을 화려한 파이 차트로 보여줍니다. 직관적인 비율 파악이 가능합니다.
   - **히스토리 및 메모 뷰**: 하단에는 최근 작업 기록 리스트가 나열되며, 어떤 내용으로 논문을 작성했는지 메모한 내용도 대시보드 내에서 바로 확인할 수 있습니다. 데이터 삭제(휴지통 버튼) 기능도 포함되어 있습니다.

5. **데스크톱 바로가기 연결**
   - 바탕화면에서 터미널 없이 아이콘 더블클릭만으로 언제든지 대시보드 접근 및 실행이 가능하도록 `ThesisTracker.desktop` 바로가기를 생성해 두었습니다.

6. **GitHub Pages 손쉬운 배포**
   - `npm run deploy` 명령어 한 번에 빌드와 GitHub 배포가 즉각적으로 이루어지는 파이프라인(`gh-pages` 모듈 활용)을 `package.json` 스크립트로 구현해두었습니다.

## 🚀 로컬 실행 방법

터미널에서 아래의 명렁어를 입력하면 로컬 개발 서버가 구동됩니다.

```bash
cd /home/garund9/projects/thesis-input-hours
npm install
npm run dev
```

브라우저가 열리면 `http://localhost:5173/` 에 접속하셔서 바로 대시보드를 사용해 보실 수 있습니다!
